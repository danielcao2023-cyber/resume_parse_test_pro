=======================================
智能简历解析与录入系统使用说明
=======================================

【一、 项目简介】
本项目是一个基于 Python 和 Gemini 大语言模型的智能简历解析工具。它可以自动读取指定文件夹中的简历文件（支持 .docx、.pdf 等常见格式），提取关键信息（如姓名、联系方式、教育背景、工作经历等），并将提取好的数据自动写入到本地的 CSV 表格中。最后，程序会根据处理结果将简历文件分类移动到对应的子文件夹中，方便管理。
最新版本加入了多线程并发处理机制，大幅提升了批量解析简历的速度，同时内置了防并发冲突的线程锁与联系方查重功能。

【二、 目录结构】
当前文件夹（/Users/caogong/Desktop/ai-test/resume_parse_test_pro）主要包含以下内容：

1. resume_parse_test_pro.py ：主程序脚本。所有的配置和逻辑都在这里。
2. resume_input/            ：简历输入文件夹。你需要把待解析的简历文件放在该目录下。
   |-- processed/           ：处理成功的简历会被自动移动到这里。
   |-- failed/              ：处理失败的简历（因网络异常、解析异常等）会被自动移动到这里。
   |-- duplicated/          ：如果在表格中检测到重复，文件会被移动到这里。
3. resume_output/           ：生成的表格数据会被自动写入到该目录下的 resumes.xlsx 文件中。

【三、 使用方法与打开方式】

1. 准备工作：
   - 确保你的电脑已安装 Python 3 环境。
   - 确保安装了程序依赖的第三方库（例如 `google-genai`, `python-docx`, `PyPDF2`, `openpyxl` 等，如果没有安装，可以在终端通过 `pip install google-genai python-docx PyPDF2 openpyxl` 进行安装）。

2. 录入简历文件：
   - 打开本项目的 `resume_input` 文件夹，将你需要处理的简历文件（Word 或 PDF）直接拖入该文件夹中。

3. 运行程序（长效监听模式）：
   - 打开 Mac 的“终端 (Terminal)”，依次运行以下三行命令：

     cd /Users/caogong/Desktop/ai-test/resume_parse_test_pro
     source ../venv/bin/activate
     python resume_parse_test_pro.py

   - 看到控制台提示“🚀 AI 招聘助手已启动 (自动监听模式) 🚀”后，程序便进入后台监听状态。此时你可以放心去做别的事情。

4. 退出程序：
   - 监听模式下，程序会自动轮询你的上传目录。如果你想停止它，只需要回到终端，在键盘上按下 `Ctrl + C`，程序即会安全退出。

5. 查看结果：
   - 运行中，控制台会实时打印各个线程解析的进度、识别到的信息以及写入表格的返回结果。
   - 运行结束后，你可以打开 resume_output 文件夹下的 resumes.xlsx 文件查看录入的数据。最新版本已支持中文表头、依据联系方式（电话/邮箱）查重，并具备自动适应列宽及自动换行的排版功能。

【四、 注意事项】

1. 网络环境（非常重要）：
   - 程序使用了 Google Gemini API。由于国内网络环境限制，在运行代码时，请务必保证你的终端/电脑处于良好的科学上网（代理）环境中。

2. 多线程与频率限制：
   - 程序默认开启了 3 个并发线程进行处理（MAX_WORKERS = 3）。这是为了能够在加速解析的同时，避免触发 Gemini 免费版 API 每分钟 15 次请求（15 RPM）的频率限制。如果你的网络较好且文件较小，刚好触发了 `429 Too Many Requests`，程序会自动等待并重试，请耐心等候。

3. 运行失败处理：
   - 如果控制台提示处理失败，对应的简历会被放进 `resume_input/failed` 文件夹中。你只需要把它们重新剪切回外层的 `resume_input` 文件夹下，检查好网络代理后，重新运行一次脚本即可重试。

4. 敏感信息保护：
   - 你的 `resume_parse_test_pro.py` 中写有你专属的大模型密钥 (GEMINI_API_KEY)。这属于个人敏感资产，请务必不要把该脚本上传到公开的 GitHub 仓库或者发送给不可信的人，以免密钥被盗刷。

5. 自动过滤：
   - 程序每次运行时会自动跳过 Mac 产生的隐藏文件（如 `.DS_Store` 或临时文件 `.~xxx.docx`），请放心使用，控制台显示跳过属于正常现象。
